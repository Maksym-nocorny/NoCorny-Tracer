## C. Регресійний smoke-тест (після кожного батчу)

Наскрізний пайплайн. L = локально (без креденшелів), R = потрібен реальний Tracer+Dropbox (офлайн-аплоуду в застосунку немає — `processRecording` гейтиться на `isSignedIn`).

**A. Happy-path (R):** увійти в Tracer → Dropbox connected → камера+мікр → старт з головного вікна → ~15с з мовленням (для B4 ще прогін рос/укр) → **пауза 3с → resume** → ~10с → **стоп** → файл фіналізується, відкривається `/v/{slug}`, thumbnail+відео вантажаться, PATCH processing→ready, локальний файл видаляється, рядок зʼявляється з AI-назвою → copy-link + trash працюють → **видалити** (серверне видалення + локальне + квота).

**B. Паритет контролів (L для стану, R для аплоуду):** старт/стоп/пауза/abort з усіх трьох точок — кнопка головного вікна, **контекст-меню в menu-bar**, **глобальні хоткеї** (⌥⇧R/P/X). Усі три дають ідентичний стан. Головна мережа регресій B2/B5.

**C. Abort (R для slug, L для файлу):** старт → abort → зупинка без збереження, локальний файл видалено, Tracer-рядок не створено.

**D. Edge-сценарії:**
- **Офлайн (R-setup, далі L):** увійти повністю → **Wi-Fi OFF** → чекати 70с (один цикл heartbeat 60с). **КРИТИЧНА перевірка B1: бібліотека НЕ стирається.** `recordings` лишаються, квота ціла. Записати офлайн → лишається `.failed`/у черзі, не зникає. Wi-Fi ON → retry успішний.
- **Дозвіл відкликано (L):** зняти Screen Recording → старт → чистий fail `.permissionDenied`, без напіввідкритого стану, UI не застряг у «recording».
- **Збій мікр посеред старту (L):** вибрати мікр, відключити, старт → відкат B2: екран-кепчер і `videoWriter` зносяться (зараз — витік).
- **Диск повний / збій запису (L):** `stopWriting()` → `nil` НЕ створює фантомний 0-байтний рядок.
- **Disconnect на сайті (R):** відключити Dropbox на tracer.nocorny.com → за ~60с застосунок чистить Dropbox-стан І стирає локальну бібліотеку (це *легітимний* `resetTracerLibraryState` — має й далі працювати після B1).
- **Зміна акаунта (R):** sign out → увійти іншим Tracer-юзером → `resetTracerLibraryState` на sign-out (щоб B не успадкував рядки A).

**Швидкий локальний цикл (без креденшелів, для B2/B5/B6):** record → pause → resume → stop із пропущеним аплоудом (signed out) → fallback на timestamp-назву, AI пропущено. Ганяти найчастіше.

**B7 smoke (L):** `bash scripts/build_dmg.sh` → змонтувати DMG → drag-install → запуск → menu bar є → «Check for Updates» влучає в appcast. Далі dry-run `release.sh` проти копії `appcast.xml` і дифф вставленого `<item>`.

## D. Запобіжна сітка — тести, яких немає (додати ПЕРЕД B1/B2)

Тест-таргета немає. Додати `.testTarget(name: "NoCornyTracerTests", ...)` у `Package.swift`. Виокремити чисті функції в library-таргет (або `@testable import`). Пріоритет — чиста логіка (без AppKit/мережі/`@MainActor`):

**Tier 1 (спершу — стережуть B1/B2/B4):**
1. **Tri-state парсинг токена/відповіді (B1):** 200→`.success`, 401/403→`.notAuthorized`, 5xx/timeout/offline→`.transportError`; канонічні `DropboxProxyToken` JSON (`{connected:false}`, валідний з `expiresAt`, битий). Найважливіший тест — фіксує розрізнення offline-vs-disconnected, що рятує від wipe.
2. **Переходи state-машини (B2):** `start` з `.recording` = no-op; `pause` з `.idle` нелегальний; `stop` з `.paused` коректно акумулює тривалість. Без AVFoundation, якщо редюсер відокремлено від side-effects.
3. **SRT-логіка `AINamingService` (B4, вже чиста):** `parseCombinedResponse`, `restoreSrtTimestamps`, `splitSegmentsByInlineTimestamps`, `splitLongSegmentBySentences`, `dominantLanguage`, `mapTimestamp`, `cleanupName`, `toRawDropboxURL`. Golden-file на реальних прикладах колапсованого SRT.

**Tier 2 (з B3/B6):** CSRF state-валідація (nil expected → FAIL), Keychain round-trip + ідемпотентність міграції, ротація/крашобезпечність логів.

**Лишається ручним (не юніт-тестити):** `SCStream`, `AVCaptureSession`, `AVAssetWriter`, `NSStatusItem`, SwiftUI-рендер, реальний Dropbox/Tracer HTTP — покрито §C.

## Крок 1. B1-api-errors: Typed tri-state results for Tracer API & Dropbox token, then fix every caller that conflates transient failure with authoritative negative
**Порядок усередині батчу:** The entire batch shares one root cause: network calls return nil/false identically for "server authoritatively says no" (disconnected / 401 / soft-deleted) and "we could not reach the server / could not understand it" (offline, timeout, 5xx, decode error).         case success(Success)
        case authoritativeNegative   // server reachable + understood; answer is a definitive "no" (HTTP 401/403/404, or {connected:false})

### 🔧 Спільний рефактор (зробити ПЕРШИМ)

The shared refactor introduces an error-classification step inside withRetry: before sleeping between attempts, inspect the thrown DropboxError.
